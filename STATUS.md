# 📊 Project Status: ESP8266 Mini Billboard

**마지막 업데이트:** 2026-02-14

## 📋 프로젝트 개요
ESP8266과 OLED 디스플레이를 활용하여 웹에서 문구와 애니메이션을 실시간으로 제어하는 '프리미엄 미니 전광판' 제작 프로젝트입니다.

## ✅ 현재 상태: 정상 작동 중
- **기기 펌웨어**: 최신 버전 업로드 완료 및 정상 구동 중.
- **네트워크**: `MIHO-2.4G` WiFi 연결 성공 (IP: `192.168.0.111`).
- **웹 서버**: 정상 작동 확인 (`curl` 및 브라우저 접속 성공).
- **디스플레이**:
  - 기본 문구("Enter Text in Web Side") 스크롤 애니메이션 작동 중.
  - 하단 상태바에 접속 주소(`HTTP://192.168.0.111`) 상시 표시.

## ✅ 해결된 문제
### Chrome에서 `ERR_ADDRESS_UNREACHABLE` 발생
- **증상**: Chrome에서 `http://192.168.0.111/` 접속 불가. Safari/curl은 정상.
- **원인**: Chrome의 **보안 DNS (Secure DNS / DNS over HTTPS)** 기능이 사설 IP를 외부 DNS 서버(Google, Cloudflare)로 질의 → 해석 불가 → 접속 실패.
- **해결**: `chrome://settings/security` → "보안 DNS 사용" 끄기.
- **교훈**:
  - 접속 불가 시 **코드 수정 전에 `curl`로 서버 정상 여부를 먼저 확인**할 것.
  - `curl` 성공 → 서버 정상 → 브라우저/클라이언트 문제 (코드 수정 불필요).
  - IoT/로컬 서버 개발 시 Chrome 보안 DNS는 꺼두는 것이 좋음.

## 🔍 Antigravity vs Claude Code 디버깅 비교

### Antigravity — "코드가 문제일 것이다"
코드만 계속 수정하는 접근:
1. Flash Erase 및 Clean Build.
2. `ESP8266WebServer` ↔ `WiFiServer` 엔진 교체.
3. 포트 변경 (80 → 8080).
4. I2C 속도 개선 및 루프 최적화.
5. AP 모드 검증.

**결과**: 코드에는 아무 문제가 없었음. 전부 헛수고.

### Claude Code — "문제 지점을 먼저 좁히자"
단계적 격리(isolation) 접근:
1. `curl http://192.168.0.111/` → **200 OK** → 서버 정상 확인.
2. 서버 문제 아님 → **브라우저 문제로 범위 축소**.
3. Safari 테스트 → 됨 → **Chrome 문제로 범위 축소**.
4. Chrome 시크릿 모드 → 됨 → **Chrome 설정 문제로 범위 축소**.
5. 보안 DNS 끄기 → **해결**.

**결과**: 코드 한 줄도 안 고치고 해결. `curl` 한 번으로 방향이 갈렸음.

### 핵심 차이

| | Antigravity | Claude Code |
|---|---|---|
| 첫 행동 | 코드 수정 | `curl`로 서버 테스트 |
| 가정 | 서버 코드가 문제 | 어디가 문제인지 모름 → 확인 |
| 방법 | 코드를 이것저것 바꿔봄 | 문제 범위를 단계적으로 축소 |
| 코드 수정 | 5회 이상 | 0회 |
| 해결 | 못함 | Chrome 보안 DNS 설정 1개 |

**교훈: 디버깅은 "고치기"가 아니라 "찾기"가 먼저다.**
